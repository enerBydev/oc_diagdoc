# 2.6.3 Build y Release

> Flujo de trabajo para transformar código Web en APK/IPA finales.

---

## Generación de Assets (Iconos/Splash)

Usar `@capacitor/assets` para generar todos los tamaños automáticamente.

1. Colocar `logo.png` (1024x1024) y `splash.png` (2732x2732) en `assets/`.
2. Ejecutar:
   ```bash
   npx @capacitor/assets generate --iconBackgroundColor '#293133' --splashBackgroundColor '#293133'
   ```

---

## Comandos de Build

### 1. Build Web (Nuxt)
Primero generar los archivos estáticos HTML/JS/CSS.
```bash
npm run generate  # Genera carpeta 'dist'
```

### 2. Sincronización (Capacitor)
Copiar `dist` a las carpetas nativas `android/` y `ios/`.
```bash
npx cap sync
```

### 3. Compilación Nativa
Abrir IDE nativo para compilar release.
```bash
npx cap open android
npx cap open ios
```

---

## Firmado (Signing)

### Android (Keystore)
> **IMPORTANTE:** Nunca subir `release.keystore` al repositorio git.

1. Generar keystore:
   ```bash
   keytool -genkey -v -keystore release.keystore -alias onlycar -keyalg RSA -keysize 2048 -validity 10000
   ```
2. Configurar `gradle.properties` (local, no git) con passwords.
3. Build Release Bundle (.aab) para Play Store.

### iOS (Provisioning Profiles)
Gestionado vía Xcode con cuenta Apple Developer Team. Requiere certificados de distribución actualizados anualmente.

---

## Versionado

Sincronizar `package.json` con nativo usando `trapeze` o manualmente.

- **version:** `2.1.0` (Marketing version)
- **buildCode:** `105` (Incremental integer para Stores)
