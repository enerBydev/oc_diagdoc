# 2.11.3 Health Checks

> "¿Estamos vivos?"

---

## Endpoint: `/api/health`

Un endpoint ligero que verifica conectividad básica.

```ts
// server/api/health.get.ts
export default defineEventHandler(async () => {
  /*
    Chequeos rápidos:
    1. DB Conectividad (Select 1)
    2. Redis/Cache (Ping)
  */
  return {
    status: 'healthy',
    timestamp: new Date(),
    version: process.env.APP_VERSION
  }
})
```

**Respuesta 200 OK:** Todo bien.
**Respuesta 503:** Servicio degradado.

---

## Monitoreo Externo

Configurar servicio externo (ej. BetterUptime) para pingear `/api/health` cada 1 minuto.
- Si falla > Alerta a Slack/SMS del equipo de guardia.
