# 2.11.1 Logging Strategy

> Logs legibles para humanos en Dev, estructurados para máquinas en Prod.

---

## Niveles de Log

| Nivel | Descripción | Ejemplo |
|-------|-------------|---------|
| **ERROR** | Fallo crítico. Requiere atención inmediata. | `PaymentGatewayTimeout` |
| **WARN** | Algo inesperado pero recuperable. | `RateLimitExceeded` |
| **INFO** | Eventos normales del ciclo de vida. | `UserLoggedIn`, `ServiceCreated` |
| **DEBUG** | Datos detallados para diagnóstico (Dev only). | `Payload: { lat: 26.5 }` |

---

## Formato Estructurado (JSON)

En producción, nunca usar `console.log` plano. Usar un logger que outputee JSON.

```json
{
  "level": "info",
  "timestamp": "2026-01-04T21:40:00Z",
  "message": "Service assigned",
  "context": {
    "serviceId": "uuid-1234",
    "operatorId": "uuid-5678"
  },
  "environment": "production"
}
```

Esto permite que herramientas como Cloudflare Logs o Datadog indexen y filtren por `context.serviceId`.

### Implementación (Consola)

En Nuxt/Nitro usamos `consola`.

```ts
import { consola } from 'consola'

consola.info('Iniciando proceso de pago...')
consola.error(new Error('Fallo conexión Stripe'))
```
