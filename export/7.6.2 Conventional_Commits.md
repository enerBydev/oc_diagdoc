# 7.6.2 Conventional Commits

> Formato estructurado para mensajes de commit.

---

## Formato

```
<type>(<scope>): <description>

[optional body]

[optional footer]
```

---

## Tipos

| Type | Descripción | Ejemplo |
|------|-------------|---------|
| `feat` | Nueva funcionalidad | `feat(citas): add calendar view` |
| `fix` | Corrección de bug | `fix(pagos): resolve webhook timeout` |
| `docs` | Documentación | `docs(api): update OpenAPI spec` |
| `refactor` | Refactorización | `refactor(domain): extract value objects` |
| `test` | Tests | `test(e2e): add payment flow` |
| `chore` | Mantenimiento | `chore(deps): upgrade Nuxt` |
| `style` | Formato (no lógica) | `style: fix linting errors` |
| `perf` | Mejora rendimiento | `perf(images): optimize lazy loading` |

---

## Scopes OnlyCar

`citas`, `pagos`, `operador`, `cliente`, `chat`, `auth`, `api`, `ui`, `deps`
