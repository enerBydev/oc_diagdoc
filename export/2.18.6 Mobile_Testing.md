# 2.18.6 Mobile Testing Strategy

> Testear en el navegador no es suficiente. El hardware real importa.

---

## Niveles de Prueba Mobile

| Nivel | Herramienta | Entorno | Qué prueba |
|-------|-------------|---------|------------|
| **L1: Web Responsive** | Chrome DevTools | Desktop | Layout básico, breakpoints |
| **L2: Mobile Web** | Playwright Mobile | CI (Linux) | Touch events, viewport real |
| **L3: Hybrid App** | Android Studio / Xcode | Emulador Local | Plugins nativos, build |
| **L4: Real Device** | BrowserStack / Device Farm | Dispositivos Reales | Hardware (GPS, Cam), Performance real |

---

## Configuración Playwright (L2)

Simulación de dispositivos móviles en CI:

```typescript
// playwright.config.ts
export default defineConfig({
  projects: [
    {
      name: 'Mobile Chrome',
      use: { ...devices['Pixel 5'] },
    },
    {
      name: 'Mobile Safari',
      use: { ...devices['iPhone 12'] },
    },
  ],
});
```

---

## Pruebas de Plugins Nativos (L3/L4)

Para probar código que usa `Capacitor`, necesitamos mockear o buildear:

### Mocking en Web (Desarrollo)

```typescript
// mocks/capacitor.ts
import { vi } from 'vitest';

vi.mock('@capacitor/geolocation', () => ({
  Geolocation: {
    getCurrentPosition: vi.fn().mockResolvedValue({
      coords: { latitude: 19.4326, longitude: -99.1332 }
    })
  }
}));
```

### Build & Test Nativo (CI)

Usamos **Maestro** (alternativa moderna a Appium) para tests E2E en APK/IPA finales.

```yaml
# .maestro/flow.yaml
appId: mx.onlycar.app
---
- launchApp
- tapOn: "Login"
- inputText: "user@test.com"
- tapOn: "Password"
- inputText: "secret"
- tapOn: "Ingresar"
- assertVisible: "Bienvenido"
```
