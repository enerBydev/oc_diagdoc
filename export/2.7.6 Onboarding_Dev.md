# 2.7.6 Onboarding Dev

> De `git clone` a `feat:` en menos de 24 horas.

---

## Día 1: El Setup

### 1. Accesos Requeridos
Solcitar al Tech Lead:
- [ ] GitHub (Repo: `onlycar_plataform`)
- [ ] Supabase (Proyecto: `onlycar-dev`)
- [ ] Cloudflare (Workers)
- [ ] Doppler (Secretos)

### 2. Instalación Local
```bash
# 1. Clonar repo
git clone git@github.com:enerBydev/onlycar_plataform.git

# 2. Setup entorno (Instala dependencias y hooks)
npm run setup

# 3. Descargar secretos
doppler login
doppler setup --project onlycar --config dev
npm run secrets:pull

# 4. Levantar servidor
npm run dev
```

### 3. Verificar Entorno
Visitar `http://localhost:3000`. Deberías ver la Landing Page.
Visitar `http://localhost:3000/api/_health`. Debería retornar `OK`.

---

## Arquitectura en 5 min

1. **Frontend:** `pages/` es tu router. Componentes en `components/` (atómicos).
2. **Backend:** `server/api/` son tus endpoints.
3. **DB:** No toques SQL directo si puedes evitarlo. Usa Supabase Client.
4. **Tipos:** Todo está en `types/`. TypeScript es estricto aquí.

---

## Tu Primera Tarea (Good First Issue)

1. Busca issues con label `good-first-issue` en GitHub.
2. Crea una rama: `git checkout -b feat/mi-primer-cambio`.
3. Haz el cambio.
4. Corre tests: `npm run test:unit`.
5. Comitea: `git commit -m "feat: mi primer cambio increíble"`.
6. Push y Pull Request.
