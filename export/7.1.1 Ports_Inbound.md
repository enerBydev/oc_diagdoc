# 7.1.1 Puertos Inbound (Primary Ports)

> Interfaces que definen los casos de uso disponibles para el mundo exterior.

---

## Concepto

Los **puertos inbound** son interfaces que representan las acciones que el sistema puede realizar. Son llamados por los adaptadores primarios (controllers, APIs).

---

## Interfaces OnlyCar

```typescript
// domain/ports/inbound/IAgendarCita.ts
interface IAgendarCita {
  execute(input: AgendarCitaInput): Promise<AgendarCitaOutput>;
}

// domain/ports/inbound/ICancelarCita.ts
interface ICancelarCita {
  execute(citaId: string, motivo: string): Promise<void>;
}

// domain/ports/inbound/IProcesarPago.ts
interface IProcesarPago {
  execute(input: ProcesarPagoInput): Promise<PagoResult>;
}

// domain/ports/inbound/IAsignarOperador.ts
interface IAsignarOperador {
  execute(citaId: string, operadorId: string): Promise<void>;
}
```

---

## Convención de Nombres

- Prefijo `I` para interfaces
- Verbo en infinitivo para la acción
- Método `execute()` estándar
