# 2.5.4 Componentes Core

> Definición técnica de los componentes base reutilizables.

---

## Estrategia de Componentes

OnlyCar utiliza una arquitectura de componentes **atómica** pero pragmática. Los "Componentes Core" (Átomos) deben ser:
1.  **Agnósticos a la lógica de negocio**: Solo reciben props y emiten eventos.
2.  **Altamente tipados**: Props definidas con Typescript interfaces.
3.  **Accesibles**: Cumplir con WCAG 2.1 (aria-labels, focus management).

---

## 1. Botones (`<BaseButton>`)

El componente de interacción principal.

### Props API
```typescript
interface ButtonProps {
  variant?: 'primary' | 'secondary' | 'outline' | 'ghost' | 'danger';
  size?: 'sm' | 'md' | 'lg';
  loading?: boolean;
  disabled?: boolean;
  icon?: string; // Nombre del icono en assets
  iconPosition?: 'left' | 'right';
  to?: string; // Si existe, renderiza <NuxtLink>
}
```

### Estados
- **Default**: Estado normal interactivo.
- **Hover/Focus**: Feedback visual inmediato (brightness-95 o ring).
- **Loading**: Muestra un spinner y deshabilita clics. Mantiene el ancho original para evitar saltos (layout shift).
- **Disabled**: Opacidad 50% y `cursor-not-allowed`.

---

## 2. Inputs (`<BaseInput>`)

Wrappers sobre elementos de formulario nativos para estandarizar estilos y errores.

### Características
- **v-model**: Soporte completo para two-way binding.
- **Error Handling**: Recibe `errorMessage` (string) y cambia el borde a rojo, mostrando el texto abajo.
- **Slots**: `prepend` y `append` para iconos o botones internos.

### Variantes
- `<BaseInputText>`
- `<BaseInputPassword>` (Toggle visibility integrado)
- `<BaseInputNumber>` (Formateo de moneda opcional)

---

## 3. Cards (`<BaseCard>`)

Contenedores de contenido con elevación y bordes consistentes.

### Diseño
- **Mobile**: Flat o borde sutil (`border-gray-100`).
- **Desktop**: Sombra suave (`shadow-sm`) en hover (`shadow-md`).
- **Padding**: Configurable (`p-4` default).

---

## 4. Feedback Components

- **`<BaseToast>`**: Notificaciones efímeras (Success, Error, Info). Integrado con un Composable `useToast()`.
- **`<BaseModal>`**: Dialogs modales accesibles (Headless UI Dialog).
- **`<BaseLoader>`**: Spinner SVG optimizado o Skeleton loader.
