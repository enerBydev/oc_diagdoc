# 7.6.1 Branch Strategy

> Estructura de ramas basada en Git Flow.

---

## Ramas Principales

| Rama | Propósito | Protegida |
|------|-----------|-----------|
| `main` | Producción estable | ✅ |
| `develop` | Integración de features | ✅ |

---

## Ramas de Soporte

| Tipo | Formato | Desde | Hacia |
|------|---------|-------|-------|
| Feature | `feature/OC-123-descripcion` | develop | develop |
| Bugfix | `bugfix/OC-456-descripcion` | develop | develop |
| Release | `release/1.0.0` | develop | main + develop |
| Hotfix | `hotfix/OC-789-descripcion` | main | main + develop |

---

## Reglas

1. **Nunca commit directo a main o develop**
2. **PRs requieren 1 aprobación**
3. **CI debe pasar antes de merge**
4. **Squash merge para features**
