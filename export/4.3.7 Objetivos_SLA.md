# 4.3.7 Objetivos SLA

> SLOs, SLIs y Error Budgets del sistema OnlyCar.

---

## Definiciones

| Término | Significado |
|---------|-------------|
| **SLA** | Service Level Agreement - Contrato con usuario |
| **SLO** | Service Level Objective - Objetivo interno |
| **SLI** | Service Level Indicator - Métrica real |
| **Error Budget** | Margen de error permitido |

---

## SLOs por Servicio

| Servicio | SLO Disponibilidad | SLO Latencia (p99) | Error Budget/mes |
|----------|-------------------|-------------------|------------------|
| API Core | 99.9% | < 500ms | 43.2 min |
| Auth | 99.95% | < 300ms | 21.6 min |
| Pagos | 99.99% | < 1s | 4.3 min |
| Realtime (Chat/Geo) | 99.5% | < 200ms | 3.6 horas |
| Storage (R2) | 99.9% | < 2s | 43.2 min |

---

## SLIs - Cómo Medimos

| SLI | Fórmula | Herramienta |
|-----|---------|-------------|
| Disponibilidad | `(requests_exitosos / total_requests) * 100` | Cloudflare Analytics |
| Latencia | `histogram_quantile(0.99, http_request_duration)` | Prometheus |
| Error Rate | `errors_5xx / total_requests` | Sentry |
| Throughput | `requests_per_second` | Grafana |

---

## Error Budget Policy

| Presupuesto Consumido | Acción |
|----------------------|--------|
| 0-50% | Operación normal |
| 50-75% | Alerta preventiva |
| 75-90% | Freeze releases, investigar |
| 90-100% | Solo hotfixes críticos |
| >100% | Incident response, postmortem |
