# 1.1.7.5.1 Validación de Empresa

> Proceso de verificación antes de agregar empresa a whitelist.

---

## Checklist de Validación

| Campo | Verificación | Fuente |
|-------|--------------|--------|
| **RFC** | Válido y activo | SAT (constancia) |
| **Razón Social** | Coincide con RFC | SAT |
| **Dominio Email** | Es de la empresa | Whois, sitio web |
| **Representante** | Autorizado | Poder notarial (opcional) |
| **Dirección fiscal** | Existe | Google Maps |

---

## Razones de Rechazo

| Código | Razón | Mensaje al Cliente |
|--------|-------|-------------------|
| `REJ-001` | RFC inválido | "El RFC proporcionado no es válido o no está activo" |
| `REJ-002` | RFC baja | "El RFC está dado de baja ante el SAT" |
| `REJ-003` | Dominio genérico | "Requerimos un email corporativo (@suempresa.com)" |
| `REJ-004` | Dominio no coincide | "El dominio no parece pertenecer a la empresa" |
| `REJ-005` | Empresa no verificable | "No pudimos verificar la existencia de la empresa" |

---

## Documentos Opcionales

Para empresas de alto volumen o casos especiales:

| Documento                      | Cuándo Solicitarlo                    |
| ------------------------------ | ------------------------------------- |
| Constancia de situación fiscal | RFC con problemas                     |
| Acta constitutiva              | Empresas nuevas                       |
| Poder notarial                 | Si firma no es el representante legal |
| Identificación oficial         | Primer contrato grande                |

## Modelo de Datos

```sql
CREATE TABLE empresas_b2b_validacion (
  id UUID PRIMARY KEY,
  empresa_id UUID REFERENCES empresas_b2b(id),
  rfc_validado BOOLEAN DEFAULT false,
  rfc_validado_at TIMESTAMPTZ,
  dominio_validado BOOLEAN DEFAULT false,
  dominio_validado_at TIMESTAMPTZ,
  constancia_situacion_url VARCHAR(500),
  resultado VARCHAR(20),  -- aprobada, rechazada, pendiente
  razon_rechazo VARCHAR(10),
  notas TEXT,
  validado_por UUID REFERENCES users(id),
  created_at TIMESTAMPTZ DEFAULT now()
);
```
