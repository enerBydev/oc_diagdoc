# 2.5.8 Micro-Interacciones y Animación

> "El movimiento da vida". Stack técnico para la Experience Layer definida en [[Proyecto OnlyCarNLD/Datos/6.0. UI-UX]].

---

## Stack de Animación

| Nivel | Herramienta | Caso de Uso |
|-------|-------------|-------------|
| **CSS Transitions** | Nativo | Hover, Focus, Active, Cambios de color. (GPU Accelerated) |
| **Vue Transition** | `<Transition>` | Entrada/Salida de elementos (v-if), Listas (v-for). |
| **Motion One** | `@motionone/vue` | Secuencias complejas, Timelines, Gestures (Drag). |

---

## Patrones de Interacción

### 1. Feedback Táctil y Visual
Cada botón o tarjeta interactiva debe tener:
*   State `hover` (Desktop): Leve elevación o cambio de brillo.
*   State `active` (Mobile): Escala al 98% (`dataset down`) para simular presión.

```css
.btn-interactive:active {
  transform: scale(0.98);
  transition: transform 0.1s ease-out;
}
```

### 2. Transiciones de Página (View Transitions API)
Nuxt Page Transtions configuradas en `bundle`:
*   Fade (Default)
*   Slide Left/Right (Navegación secuencial)
*   Crossfade Shared Element (Héroe de imagen de lista a detalle).

### 3. Carga Perceptual
*   **Skeleton Loading:** No usar spinners para contenido estructurado. Usar "shimmer" effect.
*   **Staggered Lists:** Los elementos de una lista entran con un retraso secuencial (50ms).

---

## Guidelines de Performance

1.  **Solo Transform & Opacity:** Animar `left`, `margin`, `width` causa Layout Thrashing y baja FPS. Usar `transform: translate()` y `scale()`.
2.  **Will-Change:** Usar con moderación en elementos móviles complejos.
3.  **Reduced Motion:** Respetar preferencia de usuario `@media (prefers-reduced-motion)`.
