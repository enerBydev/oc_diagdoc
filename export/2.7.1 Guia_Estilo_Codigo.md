# 2.7.1 Guía de Estilo de Código

> Reglas para mantener el código limpio y consistente automátizadamente.

---

## Configuración de Linter

El proyecto utiliza `@nuxt/eslint-config`.

```bash
npm run lint:fix
```

### Reglas Clave
1. **No `any`:** TypeScript strict mode activado.
2. **Composition API:** Usar `<script setup>` exclusivamente.
3. **Auto-imports:** No importar manualmente `ref`, `computed`, etc.

---

## Convenciones de Nomenclatura

| Elemento | Convención | Ejemplo |
|----------|------------|---------|
| **Archivos Vue** | PascalCase | `BookingCard.vue` |
| **Componentes** | Multi-word | `UserAvatar.vue` (No `Avatar.vue`) |
| **Composables** | camelCase "use" | `useGeolocation.ts` |
| **Props** | camelCase | `isValid` |
| **Eventos** | kebab-case | `update:modelValue` |
| **Variables** | camelCase | `userData` |
| **Constantes** | UPPER_SNAKE | `API_TIMEOUT` |

---

## Vue Style Guide

Seguimos las reglas Oficiales de Vue.js:

### Priority A: Essential (Error Prevention)
- Nombres de componentes multi-palabra.
- Keys en `v-for`.
- No usar `v-if` y `v-for` en el mismo elemento.

### Priority B: Strongly Recommended (Readability)
- Orden de atributos en elementos (Definición -> Renderizado -> Contenido).
- `defineProps` y `defineEmits` tipados con TypeScript.

```ts
// ✅ Correcto
const props = defineProps<{
  title: string
  isActive?: boolean
}>()
```

---

## Estructura de Componente

Orden estricto dentro de `.vue`:

1. `<script setup>`
2. `<template>`
3. `<style scoped>`
