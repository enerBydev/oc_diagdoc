# 2.14.3 Auditoría Código

> Hackear la maleza antes de que cubra el camino.

---

## Limpieza Trimestral

Cada 3 meses (Q1, Q2, Q3, Q4), se dedica un sprint parcial a:

1.  **Dead Code:** Usar herramientas (`knip`, `ts-prune`) para hallar exportaciones no usadas.
2.  **TODOs:** Revisar comentarios `// TODO` o `// FIXME`. Si tienen más de 6 meses, se arreglan o se borran.
3.  **Feature Flags:** Eliminar condicionales de features que ya están al 100% en producción.

---

## Refactor Boy Scout

La regla diaria de todo desarrollador:
*"Deja el código un poco más limpio de lo que lo encontraste."*

Si tocas un archivo para añadir una función y ves variables mal nombradas, arréglalas en el mismo PR (si es pequeño) o crea un ticket inmediato.
