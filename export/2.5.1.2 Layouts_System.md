# 2.5.1.2 Layouts System

> Elementos UI que sobreviven a la navegación.

---

## Catálogo de Layouts

En `/layouts`:

1.  **`default.vue`:**
    *   Header público, Footer con enlaces legales.
    *   Usado en Landing, Blog, Precios.
2.  **`app.vue`:**
    *   Sidebar navegación (Desktop) o TabBar inferior (Mobile).
    *   Header limpio con título de sección.
    *   **Persistent:** El sidebar no parpadea al cambiar entre `/app/dashboard` y `/app/services`.
3.  **`auth.vue`:**
    *   Centrado, sin navegación, fondo limpio.
    *   Usado en Login, Register, Recover.

---

## Transiciones Inteligentes

En `app.vue` layout, controlamos la transición de la página hija:

```vue
<template>
  <div class="layout-app">
    <SideBar />
    <main>
      <slot /> <!-- Aquí se renderiza la página -->
    </main>
  </div>
</template>

<style>
/* page-enter-active, page-leave-active definidos globalmente */
</style>
```

### Transiciones Nativas (Capacitor)
En móvil, deshabilitamos las transiciones CSS de Nuxt para usar las transiciones nativas del Webview (si es posible) o animaciones CSS extremadamente ligeras (transform, opacity) aceleradas por hardware.
