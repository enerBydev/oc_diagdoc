## Árbol de Directorios

```
OnlyCar/
├── .output/              # Build output (Cloudflare Pages)
├── .nuxt/                # Archivos generados por Nuxt
├── assets/               # Assets no compilados (CSS, Fonts)
│   ├── css/
│   └── fonts/ (Google Sans Flex/)
├── components/           # Componentes Vue (Auto-import)
│   ├── ui/               # Componentes base (Botones, Inputs)
│   ├── business/         # Componentes de negocio (BookingCard)
│   └── layout/           # Header, Footer, Sidebar
├── composables/          # Lógica reutilizable (Auto-import)
│   ├── useAuth.ts
│   └── useGeo.ts
├── layouts/              # Plantillas de estructuras de página
├── middleware/           # Guardias de navegación
├── pages/                # Rutas de la aplicación (File-system routing)
│   ├── admin/
│   ├── app/ (B2C)
│   ├── operador/
│   └── index.vue
├── plugins/              # Plugins de Vue/Nuxt (e.g. Tawk, Maps)
├── public/               # Archivos estáticos (favicon, robots.txt)
├── server/               # Backend API (Nitro)
│   ├── api/
│   └── routes/
├── utils/                # Utilidades puras TS
├── app.vue               # Punto de entrada
└── nuxt.config.ts        # Configuración principal
```

## Convenciones de Organización

### 1. Componentes
Utilizamos una estructura **Atomic Design** simplificada:
- `components/ui/`: Átomos y moléculas genéricas (botones, inputs). No tienen lógica de negocio.
- `components/business/`: Organismos que contienen lógica específica del dominio (ej. `ServicesList.vue`).
- `components/layout/`: Estructuras mayores.

### 2. Composables (`composables/`)
Toda lógica de estado y negocio que deba ser reutilizable.
- Sufijo `use` obligatorio (ej. `useBooking`).
- Auto-import habilitado.

### 3. Server (`server/`)
Endpoints de API propios de la aplicación.
- `server/api/`: Rutas prefijadas con `/api`. Retornan JSON.
- Responsable de interactuar con Supabase Admin o APIs de terceros (Stripe) de forma segura.

## Configuración Modular

Para mantener el `nuxt.config.ts` limpio, la configuración se divide si crece demasiado:

```ts
// nuxt.config.ts
export default defineNuxtConfig({
  modules: [
    '@nuxtjs/tailwindcss',
    '@pinia/nuxt',
    '@vueuse/nuxt'
  ],
  // ...config
})
```
