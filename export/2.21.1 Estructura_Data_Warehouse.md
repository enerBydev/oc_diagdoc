# 2.21.1 Estructura Data Warehouse

> **[FUTURO]** Modelo de datos optimizado para lectura y agregación, no para escritura.

---

## Modelo Dimensional (Star Schema)

Organizamos los datos en **Tablas de Hechos** (lo que pasó) y **Tablas de Dimensiones** (contexto).

### Fact Tables (Hechos)
Millones de filas, contienen métricas numéricas.
*   `fact_viajes`: `id`, `cliente_id`, `operador_id`, `monto`, `duracion`, `calificacion`.
*   `fact_pagos`: `id`, `transaccion_stripe_id`, `monto_bruto`, `comision_onlycar`.
*   `fact_logs_app`: `evento`, `pantalla`, `timestamp`.

### Dimension Tables (Dimensiones)
Contexto descriptivo para filtrar y agrupar.
*   `dim_tiempo`: Año, Mes, Día, DíaSemana, EsFeriado.
*   `dim_geo`: Ciudad, Zona, CodigoPostal.
*   `dim_usuario`: Edad, Género, FechaRegistro, FuenteAdquisicion.
*   `dim_servicio`: TipoServicio, Paquete, PrecioBase.

---

## Stack Tecnológico Potencial

1.  **Snowflake / BigQuery:** Líderes cloud, pago por consulta.
2.  **ClickHouse:** Open Source, rapidísimo para logs y eventos tiempo real.
3.  **Postgres (Réplica Analítica):** Solución inicial low-cost (Read Replica de Supabase).
