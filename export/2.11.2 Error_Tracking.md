# 2.11.2 Error Tracking

> Capturar el stack trace exacto donde ocurrió el fallo.

---

## Integración Sentry

Usamos el módulo `@nuxtjs/sentry` (o compatibilidad Nuxt 4).

### Configuración Client-Side
Captura errores de Vue, promesas rechazadas y errores de red.

```ts
// nuxt.config.ts
sentry: {
  dsn: process.env.SENTRY_DSN,
  triggers: ['onError', 'onNuxtReady'],
  clientIntegrations: {
    Replay: true // Graba video de la sesión previa al error
  }
}
```

### Configuración Server-Side (Nitro)
Captura errores en endpoints `/api` que lanzan 500.

---

## Sourcemaps

Para que Sentry muestre `OrderComponent.vue:45` en lugar de `app.js:12345`, debemos subir los sourcemaps durante el build.

> **Seguridad:** Los sourcemaps no deben ser públicos en el navegador, solo subidos a Sentry.

---

## Privacidad

Configurar `DataScrubbing` para nunca enviar:
- Passwords
- Tarjetas de Crédito
- Tokens de Auth
