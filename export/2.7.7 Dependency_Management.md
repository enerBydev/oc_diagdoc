# 2.7.7 Dependency Management

> Las dependencias son código que no escribiste pero mantienes.

---

## Reglas de Dependencias

1. **Lockfile Sagrado:** `package-lock.json` siempre se commitea.
2. **SemVer Strict:** Usar versiones exactas o caret minucioso para evitar breaking changes.
3. **Auditoría:** `npm audit` debe correr en CI.

---

## Proceso de Actualización

No actualizar "por que sí".

1. **Minor/Patch:** Semanalmente (Dependabot).
2. **Major:** Planificado trimestralmente. Requiere test de regresión completo.

### Comandos de Mantenimiento

```bash
# Ver paquetes desactualizados
npm outdated

# Actualizar interactivo
npx rad-package-installer -u

# Chequeo de vulnerabilidades
npm audit fix --force
```

---

## Análisis de Peso (Bundlephobia)

Antes de agregar una librería nueva:
1. Buscar en [Bundlephobia](https://bundlephobia.com).
2. ¿Pesa más de 50KB? ¿Hay alternativa nativa?
3. Evaluar tree-shaking support.

Ejemplo: Preferir `date-fns` sobre `moment.js` (ahorro de ~200KB).

---

## Dependabot

Configuramos GitHub Dependabot para PRs automáticos de seguridad.

```yaml
# .github/dependabot.yml
version: 2
updates:
  - package-ecosystem: "npm"
    directory: "/"
    schedule:
      interval: "weekly"
    open-pull-requests-limit: 10
```
