# 7.7 Docs-as-Code

> Documentación tratada como código: versionada, revisada y automatizada.

---

## Tipos de Documentación

### Design-Time Docs (DTD)

Documentación generada durante el desarrollo:

| Tipo | Herramienta | Output |
|------|-------------|--------|
| API Spec | OpenAPI 3.0 | `openapi.yaml` |
| Components | Storybook | Static site |
| Types | TypeDoc | HTML/MD |
| ADRs | Markdown | `docs/adr/` |

### Runtime Docs (RD)

Documentación generada en ejecución:

| Tipo | Fuente | Actualización |
|------|--------|---------------|
| API Explorer | Swagger UI | On deploy |
| Error Catalog | Sentry | Real-time |
| Metrics | Grafana | Real-time |
| Health | `/api/health` | On request |

---

## Estructura docs/

```
docs/
├── api/
│   ├── openapi.yaml          # Spec OpenAPI 3.0
│   └── postman.json          # Collection
├── adr/
│   ├── ADR-001-stack.md
│   ├── ADR-002-database.md
│   └── template.md
├── architecture/
│   ├── hexagonal.md
│   ├── ddd-contexts.md
│   └── diagrams/
└── runbooks/
    ├── deploy.md
    ├── rollback.md
    └── incident.md
```

---

## Ventajas Docs-as-Code

| Beneficio | Descripción |
|-----------|-------------|
| **Versionado** | Historial de cambios con Git |
| **Revisable** | PRs para cambios de docs |
| **Automatizable** | Generación automática |
| **Consistente** | Siempre actualizado |
| **Integrado** | Junto al código fuente |
