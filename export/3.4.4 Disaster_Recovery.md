# 3.4.4 Disaster Recovery (DR)

> Plan de contingencia ante lo impensable.

---

## Definiciones Clave

- **RTO (Recovery Time Objective)**: Tiempo máximo aceptable para volver a estar online.
    - Meta OnlyCar: < 4 horas.
- **RPO (Recovery Point Objective)**: Cantidad máxima de datos que aceptamos perder (medido en tiempo).
    - Meta OnlyCar: < 1 minuto (gracias a PITR).

---

## Escenarios de Emergencia

### Nivel 1: Corrupción de Datos Lógica
*Causa*: Un script erróneo borró usuarios o una migración rompió la integridad.
*Acción*: Usar PITR de Supabase para restaurar la DB a 5 minutos antes del incidente.
*Impacto*: Downtime de 15-30 minutos mientras se restaura.

### Nivel 2: Falla de Región (Supabase/Cloudflare)
*Causa*: El datacenter donde vive Supabase se cae.
*Acción*:
1.  Activar modo mantenimiento en Frontend (Página estática informativa).
2.  Si la caída > 4 horas, provisionar nueva instancia en `us-east-1`.
3.  Restaurar desde el último Backup Externo (R2).
*Impacto*: Pérdida de datos desde el último backup externo. (Alto impacto).
