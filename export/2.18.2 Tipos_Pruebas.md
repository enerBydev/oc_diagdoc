# 2.18.2 Tipos de Pruebas

> Tipos de pruebas por módulo.

---

## Pruebas de Autenticación

| Caso | Tipo | Validación |
|------|------|------------|
| Login OAuth | E2E | Redirect correcto, sesión creada |
| Login Email/Pass | Integration | Hash verificado, token generado |
| Detección tipo usuario | Unit | Dominio → rol correcto |
| MFA requerido | E2E | Pantalla MFA mostrada |
| Logout | Integration | Sesión invalidada |

---

## Pruebas de Pagos

| Caso | Tipo | Validación |
|------|------|------------|
| Payment Link | E2E | Stripe checkout funcional |
| Webhook payment_intent.succeeded | Integration | DB actualizada |
| Pago fallido | Integration | Notificación enviada |
| Suscripción | E2E | Cobro recurrente configurado |
| MercadoPago Point | Integration | Terminal conectado |

---

## Pruebas de Facturación

| Caso | Tipo | Validación |
|------|------|------------|
| CFDI auto (Stripe→Gigstack) | Integration | UUID SAT recibido |
| Autofactura | E2E | Portal accesible, datos capturados |
| Factura global | Integration | Ventas incluidas correctamente |
| Webhook cfdi.created | Integration | PDF/XML descargados a R2 |

---

## Pruebas de Contratos

| Caso | Tipo | Validación |
|------|------|------------|
| Generación PDF | Unit | PDF generado correctamente |
| Envío a Mifiel | Integration | document_id recibido |
| Webhook document.signed | Integration | PDF firmado descargado |
| Notificación cliente | E2E | Email recibido |
