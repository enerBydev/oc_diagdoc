# 2.4.2 Env Staging

> Ambiente de Pre-Producción / QA.

---

## Propósito
Validar cambios exactos como se verán en producción antes de hacer merge a `main`.
Se despliega automáticamente en Vercel/Cloudflare Pages al hacer PR.

## Diferencias con Prod
- **Base de Datos**: Usa una instancia separada o un esquema `staging` en Supabase.
- **Correos**: Los correos salientes NO se envían a usuarios reales (o se envían solo a whitelist `@onlycar.mx`).
- **Robots**: `X-Robots-Tag: noindex` forzado para evitar SEO duplicate content.

```bash
# Vercel System Env
NUXT_PUBLIC_SITE_URL=https://staging.onlycar.mx
NUXT_PUBLIC_ENV=staging
```
