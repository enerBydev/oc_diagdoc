# 2.4.1 Env Development

> Archivo `.env` para desarrollo local (`npm run dev`).

---

## Ejemplo Base

Copiar `.env.example` a `.env`.

```bash
# App
NUXT_PUBLIC_SITE_URL=http://localhost:3000

# Supabase (Proyecto de Prueba)
NUXT_PUBLIC_SUPABASE_URL=https://proyectoprueba.supabase.co
NUXT_PUBLIC_SUPABASE_KEY=ey... (Anon Key)
SUPABASE_SERVICE_KEY=ey... (Service Key)

# Stripe (Test Mode)
STRIPE_SECRET_KEY=sk_test_...
STRIPE_WEBHOOK_SECRET=whsec_...

# Flags
NUXT_PUBLIC_DEV_MODE=true
```

## Mocking
En desarrollo, algunas integraciones pueden estar "mockeadas" para trabajar offline.

- `MOCK_PAYMENTS=true`: Simula pagos exitosos sin llamar a Stripe.
- `MOCK_GEO=true`: Usa coordenadas fijas (Oficina OnlyCar) en lugar de GPS real.
