# 2.9 Arquitectura Backend (Nitro)

> Lógica de servidor ejecución en el Edge (Cloudflare Workers) o Serverless.

---

## Visión General

Aunque OnlyCar usa Supabase como "Backend-as-a-Service", existe lógica de negocio crítica que no puede correr en el cliente (por seguridad) ni en la base de datos (por complejidad). Esta lógica reside en el servidor de Nuxt (**Nitro**).

## Responsabilidades del Backend (Nitro)

1. **Orquestación:** Coordinar pagos (Stripe) con base de datos.
2. **Privacidad:** Manejar llaves secretas (`STRIPE_SECRET_KEY`) lejos del cliente.
3. **Complejidad:** Algoritmos de asignación (Matching) y cálculo de rutas.
