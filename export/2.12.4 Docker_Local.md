# 2.12.4 Docker Local

> Reproducir la infraestructura de producción en tu laptop.

---

## ¿Por qué Docker?

Aunque usamos servicios gestionados (SaaS), desarrollar contra la nube puede ser lento o caro. Usamos Docker para levantar réplicas locales.

---

## Stack Local (Docker Compose)

```yaml
# docker-compose.yml
version: "3.8"
services:
  # Supabase Local (DB, Auth, Storage, Realtime)
  supabase:
    image: supabase/studio:latest
    ports: ["54322:5432"]
    environment:
      POSTGRES_PASSWORD: ${DB_PASS}

  # Redis (Cache Local)
  redis:
    image: redis:alpine
    ports: ["6379:6379"]

  # Mailpit (Email Testing)
  mailpit:
    image: axllent/mailpit
    ports: ["8025:8025", "1025:1025"]
```

---

## Comandos Útiles

```bash
# Iniciar stack completo
npm run docker:up

# Ver logs
npm run docker:logs

# Limpiar todo (Reset DB)
npm run docker:nuke
```

---

## Supabase CLI vs Docker

Recomendamos usar **Supabase CLI** para desarrollo estándar, ya que maneja los contenedores por ti de forma optimizada.

```bash
# Iniciar Supabase localmente
npx supabase start

# Ver estado
npx supabase status

# Parar
npx supabase stop
```

El `docker-compose.yml` se reserva para servicios adicionales como Redis custom o microservicios de Python (IA) que no son parte del stack estándar de Supabase.
