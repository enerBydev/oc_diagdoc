# 7.4.2 Strategy Pattern

> Permite intercambiar algoritmos en tiempo de ejecución.

---

## Strategies OnlyCar

| Strategy | Variantes | Uso |
|----------|-----------|-----|
| `PricingStrategy` | Standard, Dynamic, Corporate | Cálculo de precios |
| `NotificationStrategy` | Push, Email, SMS | Envío de notificaciones |
| `PaymentStrategy` | Stripe, MercadoPago | Procesamiento de pagos |
| `RoutingStrategy` | Nearest, Rating, Load | Asignación de operadores |

---

## Ejemplo: Pricing

```typescript
interface IPricingStrategy {
  calculate(base: number, ctx: PricingContext): number;
}

class DynamicPricingStrategy implements IPricingStrategy {
  calculate(base: number, ctx: PricingContext): number {
    const demandFactor = this.getDemandFactor(ctx.timestamp);
    const distanceFactor = this.getDistanceFactor(ctx.distance);
    return base * demandFactor * distanceFactor;
  }
}

// Uso en servicio
class PricingService {
  constructor(private strategies: Map<string, IPricingStrategy>) {}
  
  calculate(tipo: string, base: number, ctx: PricingContext): number {
    return this.strategies.get(tipo)!.calculate(base, ctx);
  }
}
```
